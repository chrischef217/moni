# MONI Control Tower Dashboard V1 — 2026-07-23

## 목적
MONI 로그인 후 첫 화면을 개별 업무 메뉴가 아닌 **돈 중심의 통합 경영 Control Tower**로 전환한다.

## V1 실제 연결 데이터
### 판매관리
- 이번 달 확정 판매 건수
- 이번 달 매출 합계
- 현재 `미입금/일부입금` 상태 판매 관리 대상 금액
- 미입금/일부입금 판매 리스트

주의: 현재 판매관리에는 실제 부분입금 금액과 입금예정일이 별도 기록되지 않으므로, `payment_status != paid`인 판매는 판매 총액이 관리 대상 금액에 포함된다. 이 값을 정확한 매출채권 잔액으로 오해하지 않는다.

### 생산관리
기존 `/api/moni/production-dashboard` 계산을 그대로 재사용한다.
- 오늘까지 생산계획 대비 실적/달성률
- 생산 로스율 및 확인단가 기준 로스 금액 영향
- 향후 14일 원재료 부족 종수
- 위험 작업지시 건수
- 부족분 확인단가 기준 예상 구매액
- 생산 경고 목록

## V1에서 의도적으로 숫자를 만들지 않는 항목
아래는 아직 원천 데이터 구조가 확정되지 않았으므로 `설정 필요 / 연결 대기`로 표시한다.
- 영업 목표매출 및 목표 달성률
- 정확한 거래처 수금 D-Day / 연체일수
- 실제 부분입금 후 남은 매출채권 잔액
- 현재 현금/예상 현금흐름
- 세금 예상액
- 판매 수익률/제품별 실제 이익

## 첫 화면 구조
1. MONEY FIRST — 이번 달 매출 / 미입금 관리 대상 / 목표매출 연결상태 / 현금·세금 연결상태
2. ACTION QUEUE — 미입금 및 생산 위험 중 즉시 확인할 항목
3. BUSINESS FLOW — 영업 목표 → 판매 실적 → 회수 → 생산 흐름
4. PRODUCTION & COST — 돈에 영향을 주는 생산 KPI
5. COLLECTION FOCUS — 미입금/일부입금 판매 리스트
6. MONI BRIEFING — 현재 내부 시스템 경고. 향후 Intelligence Board로 확장
7. NEXT DATA CONNECTION — 수금·미수금 → 다중 판매규격 → 영업 목표매출 순서

## 갱신
- 최초 로딩 시 실데이터 조회
- 60초 자동 갱신
- 사용자가 즉시 새로고침 가능

## 내비게이션
- `/` = MONI Control Tower
- 기존 `AdminDashboard` 기반 화면은 `/?legacy=1`에서 계속 유지한다.
- 글로벌 사이드바에 `통합 대시보드 > 경영 Control Tower`를 정식 카테고리로 추가한다.
- 기존 생산관리의 DOM 기반 내부화면은 글로벌 사이드바에서 `/?legacy=1`로 이동 후 기존 target을 실행하여 보호한다.

## 다음 개발 우선순위
1. 판매관리 수금·미수금/매출채권: 입금예정일, 실제 입금내역, 부분입금 잔액, D-Day/연체
2. 제품 다중 판매규격/다중 단가
3. 영업관리 목표매출
4. 회계·세무관리 현금흐름/지급/세금
5. 위 데이터들을 Control Tower에 순차 연결

## PMO 원칙
- 대시보드는 화려한 요약화면이 아니라 **오늘 무엇을 해야 돈을 벌고 돈을 놓치지 않는지 알려주는 통제화면**으로 유지한다.
- DB에 없는 값은 추측하지 않는다.
- 각 모듈에서 이미 검증된 계산 로직을 우선 재사용한다.
