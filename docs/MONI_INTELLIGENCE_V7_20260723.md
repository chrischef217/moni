# MONI Intelligence V7 — 결정형 경영 우선순위 엔진

## 목적
MONI에 연결된 실제 경영 데이터를 읽고 `오늘 무엇을 먼저 해야 하는가`를 자동 정렬한다.

V7의 Intelligence는 생성형 AI의 추측이 아니다.
구조화된 원천데이터와 문서화된 판정 규칙만 사용한다.

## 입력 데이터
- 생산 대시보드
- 수금·미수금 V3
- 영업 목표매출 V5
- 현금흐름·세무 V6

## 판정 레벨
1. `critical` — 즉시 조치
2. `high` — 높은 우선순위
3. `attention` — 주의
4. `data` — 데이터 보완
5. `good` — 운영위험 없음

운영위험과 데이터 누락을 같은 것으로 취급하지 않는다.

## 핵심 판정 규칙
### 즉시 조치
- 연체 미수금이 1건 이상
- 향후 30일 예정자금 순증감이 0원 미만
- 향후 작업지시 중 재고부족 위험 또는 부족 원재료 존재

### 높은 우선순위
- D-3 ~ D-Day 수금예정 미수금 존재
- 월 목표 부족액이 존재하며, 특히 이번 달 종료예정 파이프라인 원금액으로도 부족액을 덮지 못하는 경우
- 생산 로스율 2% 이상 — 기존 생산 대시보드 경고기준 재사용

### 데이터 보완
- 회사 월 목표매출 미설정
- 미수금 입금예정일 미설정
- 계좌/현금함 미등록, 잔액 미등록, 7일 초과 Snapshot
- 생산 사용 원재료 단가 미등록
- 열린 영업기회의 예상종료일 미설정
- 지급완료 정산의 실제 지급일 미기록

## 영업 파이프라인 원칙
CRM에는 확률 필드가 없다.
따라서 리드 10%, 제안 50%, 협상 80% 같은 가중치를 만들지 않는다.

목표 부족액과 `close_date`가 이번 달인 열린 영업기회의 `expected_amount` 원금액을 단순 비교할 뿐이며, 이를 실제 예상매출 또는 확정매출로 표현하지 않는다.

## 현금 원칙
- 매출 ≠ 입금
- 원가 ≠ 현금지출
- 예정 순증감 ≠ 은행잔고
- 은행잔고는 V6 Balance Snapshot만 사용

## 세무 원칙
V7은 세무 참고값을 위험판정의 확정세금으로 사용하지 않는다.
세무 신고액, 공제요건, 납부기한은 별도 검토영역이다.

## 출력
각 Intelligence item은 다음을 반드시 포함한다.
- severity
- priority
- title
- summary
- evidence
- action
- href
- source
- rule

사용자는 `근거` 버튼으로 왜 해당 판정이 나왔는지 확인할 수 있다.

## 화면
`/intelligence`
- TOP ACTION
- 경영 행동 우선순위
- 데이터 품질 보완
- 판정 규칙/근거
- 엔진 투명성

메인 Control Tower에는 우측 하단 `MONI Intelligence` Quick Access를 두어 현재 최우선 판정과 긴급 항목 수를 표시한다.

## 갱신
- Intelligence 화면 60초 자동 재판정
- Control Tower Quick Access 60초 자동 재판정
- 수동 `다시 판정` 지원

## V7 한계
- 아직 카카오 알림을 보내지 않는다. 사용자 지시에 따라 외부 메신저 연동은 후순위다.
- 생성형 모델이 경영판단 사실을 만들어내지 않는다.
- 은행/세무 외부시스템 자동연동이 없으므로 미등록 사실을 추측하지 않는다.

## 다음 단계
V7 운영 후 규칙별 오탐/미탐을 실제 사용으로 검증한다.
그 이후 생성형 모델은 구조화된 V7 evidence를 설명하거나 문장화하는 보조 레이어로만 검토한다.
