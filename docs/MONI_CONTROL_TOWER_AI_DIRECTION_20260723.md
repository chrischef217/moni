# MONI Control Tower & AI Character Direction — 2026-07-23

## PMO 최상위 목적
MONI의 최우선 목적은 **회사의 돈을 벌고, 돈을 놓치지 않고, 더 많이 벌 수 있도록 의사결정을 돕는 것**이다.
기능별 메뉴는 독립 시스템이 아니라 하나의 경영 데이터 흐름으로 연결한다.

## 1. 돈 우선 KPI 체계
### 회계·세무관리
- 현재 재무상태 진단
- 현금흐름/세금/비용/수익성 진단
- 실제 경영 상태를 영업·판매·생산 의사결정에 전달

### 영업관리
- **목표매출 설정 기능 필수**
- 기간별/담당자별/거래처군별 목표를 설정할 수 있도록 설계
- 목표 대비 파이프라인, 예상매출, 확정매출, 부족액을 추적
- 목표 달성을 위해 필요한 신규영업/견적/계약 수준을 역산할 수 있도록 확장

### 판매관리
- 판매실적, 수익률, 수금/미수금, 연체, 거래처별 채권을 핵심 지표로 관리
- 목표매출 대비 실제 판매성과와 수익성을 진단
- 목표 부족 시 추가 판매 필요량과 우선 거래처/제품을 제안할 수 있도록 확장
- 판매관리 최우선 기능은 **수금·미수금 및 매출채권 관리**

### 생산관리
- 단순 생산실적을 넘어서 영업/판매/재무에서 전달된 선행지표를 반영
- 목표매출 달성에 필요한 예상 생산량
- 향후 판매계획 기반 원재료 부족 위험
- 고수익/고수요 제품의 생산 우선순위
- 수금/판매 흐름에 따른 과잉생산 위험 등 경영형 생산 KPI를 단계적으로 연결

## 2. MONI 전체 대시보드 = 첫 화면
MONI 로그인 후 첫 화면은 개별 모듈이 아니라 **전체 경영 Control Tower Dashboard**로 한다.

대시보드는 최소 다음 축을 한 화면에서 통제한다.
- 돈: 현금, 매출, 미수금, 연체, 예상입금, 비용, 세금
- 목표: 목표매출, 현재 달성률, 예상 달성률, 부족매출
- 영업: 파이프라인, 견적, 계약 가능성, 담당자 성과
- 판매: 실매출, 수익률, 주요 거래처, 주요 제품, 채권 위험
- 생산: 생산계획, 실제 생산, 원재료 위험, 목표매출 연동 생산 필요량
- 경고: 돈/영업/판매/생산에서 즉시 조치가 필요한 항목

대시보드는 단순 조회 화면이 아니라 각 모듈로 바로 이동하고 우선조치를 실행하는 통제 화면이어야 한다.

## 3. MONI AI는 별도 대카테고리가 아니다
AI 채팅을 사이드바의 독립 메뉴로 두는 것이 핵심 구조가 아니다.
**MONI AI는 모든 화면 위에 항상 존재하는 Global Floating Agent**로 구현한다.

## 4. MONI 캐릭터
- MONI 전용 캐릭터를 만든다.
- 모든 페이지 우측 하단 등에 Floating Character 형태로 항상 표시한다.
- 정적인 아이콘이 아니라 가벼운 실시간 반응/모션을 사용한다.
- 알림 상태, 긴급도, AI 작업 상태 등에 따라 표정/동작/상태를 바꿀 수 있도록 확장한다.
- 사용자가 클릭 가능한 대상임을 자연스럽게 인지할 수 있어야 한다.
- 새 알림이 없을 때는 말풍선 예: **“MONI에게 무엇이든 물어보세요.”** 를 표시한다.
- 캐릭터 클릭 시 Floating AI Chat Popup이 열린다.

## 5. AI의 역할
MONI AI는 단순 Q&A 챗봇이 아니다.
MONI 전체 Context와 각 업무 모듈의 API/데이터를 기반으로 다음 역할을 수행한다.

### 내부 경영 분석
- 재무/세무/영업/판매/생산 데이터를 연결해 선제 분석
- 목표매출 미달 가능성, 미수금/연체, 수익성 악화, 생산 부족/과잉 등을 먼저 탐지
- 사용자가 묻기 전에 필요한 행동을 제안

### 돈을 벌기 위한 제안
- 추가 매출 기회
- 고수익 제품/거래처 우선순위
- 영업 활동 강화 포인트
- 가격/마진/수익성 개선 후보
- 생산 우선순위 조정
- 시스템 내부 데이터만으로 찾을 수 없는 마케팅/비즈니스 기회도 외부 정보와 결합해 제안

### 외부 정보 수집
관리자가 설정한 주기(예: N분 간격)에 따라 외부 정보를 수집/분석할 수 있도록 설계한다.
예시:
- 국내 식약처/정부/규제 뉴스
- 식품·원재료·유통 관련 이슈
- 시장/경쟁사/마케팅 트렌드
- 해외 비즈니스/수출 관련 이슈
- 날씨 등 생산·물류·영업에 영향을 줄 수 있는 정보
- 기타 회사의 돈/리스크/기회에 영향을 줄 수 있는 글로벌 정보

단순 뉴스 수집이 아니라 **MONI 사업과의 관련성, 영향도, 긴급도, 예상 금액 영향**을 분석해서 전달하는 방향을 우선한다.

## 6. MONI 말풍선 알림
MONI 캐릭터는 중요한 정보를 말풍선 형태로 먼저 전달한다.
예시:
- “오늘 입금 예정 3건, 총 5,800,000원입니다.”
- “A거래처 2,300,000원이 5일 연체되었습니다.”
- “현재 영업 흐름이면 월 목표매출이 약 12% 부족할 가능성이 있습니다.”
- “이번 주 판매 추세 기준으로 다음 주 특정 제품 생산량이 부족할 가능성이 있습니다.”
- “새로운 식약처 관련 공지가 확인됐습니다. 현재 제품에 영향을 줄 가능성이 있습니다.”

## 7. MONI News / Intelligence Board
AI가 생성한 선제 알림·뉴스·외부 이슈는 일회성 말풍선으로 끝내지 않는다.
**별도 누적 게시판**에 영구적으로 쌓아 추적 가능하게 한다.

### 메인 대시보드
- 하단에 작은 카드형 최신 게시물 리스트 배치
- 중요도/카테고리/시간/읽음 여부 표시
- 클릭 시 전체 Intelligence Board를 팝업 또는 전용 뷰로 연다.

### 전체 게시판
- 전체 리스트 조회
- 카테고리/중요도/기간/읽음 상태 검색·필터
- 게시물 클릭 시 상세 내용 표시
- 관련 원문/이슈 링크 기록
- 생성시각/확인시각 기록
- 조회수(확인 횟수) 카운트
- 관련 MONI 내부 데이터/메뉴로 바로 이동 가능하게 확장
- 동일 이슈 중복 수집 방지 및 후속 업데이트 연결 구조 필요

## 8. 관리자 설정
관리자가 최소 다음을 설정할 수 있도록 한다.
- 외부 정보 수집 주기
- 알림 주기
- 수집할 정보 카테고리
- 중요도 기준
- 알림을 받을 시간대/조용한 시간
- 말풍선 노출 빈도
- 데이터 소스/API 활성화 여부

## 9. AI 자율성의 경계
“스스로 생각하고 발전한다”는 방향은 **선제 분석·추천·학습된 운영규칙 개선**을 의미한다.
다만 실제 판매/재고/회계/세무/삭제/금액 변경 등 중요한 시스템 데이터는 AI가 사용자 승인 없이 임의 변경하지 않는다.

- 조회/분석/알림/추천: 자동 가능
- 등록/수정: 영향도에 따라 승인 후 실행
- 삭제/재고보정/회계 중요 변경: 강한 승인 필수

AI의 발전은 모델 자체를 임의 변경하는 것이 아니라 결과/피드백/검증을 바탕으로 Prompt, SOP, Knowledge, Tool, Capability Package를 개선하는 방식으로 한다.

## 10. 외부 메신저 알림 = MONI가 사용자를 찾아가는 채널
MONI는 한국 두배의 웹 기반 운영 시스템이므로 사용자가 브라우저를 열고 있지 않아도 중요한 사항을 모바일로 전달할 수 있어야 한다.
따라서 내부 말풍선/Intelligence Board와 별도로 **External Notification Gateway**를 둔다.

### 한국 두배 기준 1차 채널
- **1차: 카카오 알림톡**
- 카카오 알림톡은 결제·주문·배송 등과 같은 개인화된 정보성 메시지를 API로 발송할 수 있는 카카오비즈니스 채널이며, MONI의 수금·지급·기한·필수입력 누락 알림에 우선 적용한다.
- 실제 연동은 카카오비즈니스 공식 딜러사를 통한 알림톡 API 사용을 기본 전제로 한다.
- 수신 실패나 카카오톡 수신 불가 상황에 대비해 **SMS/LMS를 보조 Fallback 채널**로 설계한다.
- 이메일·Web Push 등은 Notification Gateway의 추가 채널 어댑터로 확장할 수 있다.
- LINE은 두배 MONI의 기본 채널로 사용하지 않는다. 향후 태국 법인 등 별도 해외 시스템에서 필요할 경우에만 별도 검토한다.

### 카카오 양방향 MONI 확장
- 선제 운영 알림은 카카오 알림톡을 우선 사용한다.
- 사용자가 카카오톡 안에서 MONI에게 질문하고 답을 받는 기능은 알림톡 자체와 분리하여 **카카오톡 채널 챗봇 + 외부 Skill API** 방식으로 확장 검토한다.
- 카카오톡 채널 챗봇은 외부 API와 데이터를 주고받을 수 있으므로 MONI 조회 Tool과 연결하는 구조를 목표로 한다.
- 예: “오늘 받을 돈?”, “연체 거래처 알려줘.”, “오늘 생산 입력 빠진 거 있어?”
- 조회/분석은 바로 답변할 수 있지만 실제 등록/수정/삭제는 기존 AI 승인 원칙을 그대로 적용한다.

### 전달 대상 중요 이벤트
- 거래처 대금 입금예정 D-Day / D-1 / D-3
- 미수금/연체 발생 및 연체기간 증가
- 매입처 지급예정 D-Day 및 지급 지연
- 현금 부족/예상 현금흐름 위험
- 생산 완료 후 실적 미입력
- 작업지시가 진행 중인데 완료/수량/LOT 등 핵심 데이터 미입력
- 원재료 단가, 판매단가, 결제예정일 등 필수값 누락으로 계산/예측/정산 로직이 불완전한 상태
- 목표매출 달성 위험
- 재고/원재료 부족 위험
- 세금/신고/지급 기한
- 기타 MONI가 중요하다고 판정한 돈/리스크/기회 이벤트

### 알림 우선순위
- Critical: 즉시 발송 + 미확인 시 재알림/상향
- High: 정해진 짧은 주기 내 발송
- Normal: 일괄 브리핑 또는 설정 주기 발송
- Info: Intelligence Board 적재 중심, 필요 시 요약 발송

### 알림 메시지 원칙
단순히 “문제가 있습니다”라고 보내지 않는다.
항상 가능한 범위에서 다음을 포함한다.
- 무엇이 문제인지
- 금액/기한/영향
- 왜 중요한지
- 지금 해야 할 행동
- MONI 관련 화면으로 이동하는 Deep Link

예:
- “A거래처 2,300,000원 입금예정일이 오늘입니다. 아직 입금 기록이 없습니다. [수금관리 열기]”
- “공룡빅맥소스 생산은 완료 예정시각을 지났지만 생산완료 입력이 없습니다. 현재 재고·원가·판매가능수량 계산이 불완전합니다. [생산완료 입력]”
- “원재료 3종의 매입단가가 비어 있어 이번 달 제품 수익률 계산 정확도가 떨어집니다. [누락정보 입력]”

### 확인/해결 상태 추적
외부 알림도 일회성 문자로 끝내지 않는다.
각 알림은 MONI 내부 Alert/Event 레코드와 연결하고 최소 다음 상태를 가진다.
- 신규
- 발송
- 확인
- 조치중
- 해결
- 무시/보류

사용자가 메신저 또는 MONI에서 문제를 확인·해결하면 동일 이벤트의 반복 알림을 중단한다.

### 재알림/에스컬레이션
- 같은 문제를 매 주기마다 무작정 반복 발송하지 않는다.
- D-Day, 연체일수, 금액, 긴급도에 따라 재알림 규칙을 적용한다.
- 사용자가 확인하지 않았고 금액/위험이 큰 경우 알림 강도를 높인다.
- 동일 이벤트 중복 생성 방지용 dedupe key가 필요하다.

## 11. 개발 우선순위
1. 판매관리의 수금·미수금/매출채권 + 판매 대시보드
2. 판매관리 다중 판매규격/다중 단가 및 거래처별 조건
3. 영업관리 목표매출/파이프라인/달성률
4. 회계·세무관리 경영진단 및 현금/비용/세금 KPI
5. 생산관리 경영 연동 선행지표 추가
6. 전체 Control Tower Dashboard
7. Global MONI Character + Floating AI Chat
8. Proactive Intelligence/News Engine + Intelligence Board
9. External Notification Gateway + 카카오 알림톡/SMS Fallback
10. 카카오톡 채널 챗봇 + MONI AI 조회 Tool 연동
11. 각 모듈 Tool/API를 AI Agent에 연결하고 승인체계 적용

## PMO 판단
MONI는 단순 ERP/공장관리 시스템이 아니라 **돈 중심의 AI Business Operating System**으로 발전시키는 방향을 공식 기준으로 한다.
MONI의 중요한 차별점은 사용자가 시스템을 열어보기를 기다리는 것이 아니라, **돈·기한·데이터 누락·위험·기회를 스스로 탐지해 웹 내부와 외부 메신저 양쪽에서 사용자를 찾아가는 것**이다.
