# 판매관리 V1 — MONI

## 1. 메뉴 위치

판매관리는 영업관리 하위가 아니라 별도 대카테고리로 운영한다.

하위 메뉴:

1. 거래처 관리
2. 판매 등록
3. 거래명세표
4. 판매 통계
5. 세금계산서

세금계산서는 V1에서 메뉴와 준비 중 화면만 제공한다. 국세청 전송이나 외부 ASP 연동은 구현하지 않는다.

## 2. 데이터 공유 원칙

- 거래처 마스터는 기존 `sales_clients`를 그대로 사용한다.
- 담당 프리랜서는 기존 `business_people`의 `sales_freelancer`를 사용한다.
- 판매 제품은 기존 `products` 활성 항목에서 선택한다.
- 영업관리와 판매관리에서 거래처를 중복 등록하지 않는다.

## 3. 판매 흐름

```text
거래처 등록
→ 담당 영업 프리랜서 지정
→ 판매 등록
→ 거래명세표 출력
→ 입금상태 관리
→ 거래처·제품·프리랜서별 판매 통계
```

## 4. 거래명세표

- 번호 형식: `DB-YYYYMMDD-NNN`
- 공급가액, 부가세, 합계금액 자동 계산
- 브라우저 인쇄 기능으로 출력 및 PDF 저장
- 취소된 판매 건은 출력 불가
- 수정·취소 전 기존 데이터는 `sales_order_history`에 스냅샷 저장

## 5. 현재 권한

V1은 관리자 화면으로 운영한다. 프리랜서 본인 계정에서 담당 거래처와 판매 내역을 직접 관리하는 기능은 후속 범위로 분리한다.

## 6. 관련 테이블

- `sales_clients`
- `sales_orders`
- `sales_order_items`
- `sales_order_history`
- `business_people`
- `products`

## 7. 계산 기준

- 품목 공급가액 = 수량 × 단가
- 판매 공급가액 = 품목 공급가액 합계
- 부가세 = 공급가액 × 선택 세율
- 합계금액 = 공급가액 + 부가세
- 판매 통계는 `status='confirmed'`인 판매 건만 반영
- 취소·작성중 판매 건은 통계에서 제외
