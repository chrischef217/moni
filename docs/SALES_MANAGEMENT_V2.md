# MONI 판매관리 V2

## 목적
생산관리 Product Master를 판매관리에서 재사용하고, 제품·거래처·영업 프리랜서·실제 판매·월 정산을 하나의 데이터 흐름으로 연결한다.

## 확정 구조
1. 제품 판매설정
   - 생산관리 `products`를 별도 복제하지 않고 참조한다.
   - 판매 가능 여부, 기본 판매단위(kg/EA/BOX), 개별중량(g), 카톤박스 입수량, 기본 판매단가, MOQ를 관리한다.
   - INLINE 반제품은 판매 가능으로 설정할 수 없다.
2. 거래처 관리
   - `sales_clients`를 거래처 Master로 사용한다.
   - 거래처별 영업 프리랜서는 0명, 1명, 여러 명 연결 가능하다.
3. 거래처별 판매조건
   - `거래처 × 제품` 단위로 실제 판매단위, 납품단가, MOQ를 관리한다.
   - 해당 거래처에 연결된 영업 프리랜서별 `영업 정산단가(원/kg)`를 별도로 관리한다.
4. 판매 등록
   - 거래처를 먼저 선택한 뒤 해당 거래처에 설정된 제품만 판매등록한다.
   - 판매단위와 납품단가는 거래처별 판매조건에서 자동 적용한다.
   - EA/BOX 판매도 제품 판매설정의 중량과 카톤 입수량을 이용해 kg으로 환산한다.
5. 영업 프리랜서 월 정산
   - 판매 확정 시 실제 판매 kg, 당시 담당자, 당시 원/kg 정산단가를 Snapshot으로 저장한다.
   - 이후 판매조건이 변경돼도 과거 정산액은 변경되지 않는다.
   - 취소 판매는 월 정산에서 제외한다.

## 데이터 테이블
- `sales_product_settings`
- `sales_client_people`
- `sales_client_product_terms`
- `sales_client_product_agents`
- `sales_order_item_settlements`
- 기존 `sales_clients`, `sales_orders`, `sales_order_items`, `business_people`, `products` 재사용

## 화면 순서
- 제품 판매설정
- 거래처 관리
- 거래처별 판매조건
- 판매 등록
- 거래명세표
- 영업 정산서
- 판매 통계
- 세금계산서
- 수출 관리

## 기존 구조 보호
- 기존 판매 V1 데이터는 삭제하지 않는다.
- 기존 `assigned_person_id`와 `commission_rate` 필드는 호환용으로 유지한다.
- 신규 정산 기준은 `% 매출 커미션`이 아니라 `거래처 × 제품 × 프리랜서별 원/kg`이다.
- 수출 관리는 별도 샘플 시스템 검토 후 확장한다.
